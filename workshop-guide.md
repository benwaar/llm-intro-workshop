# Workshop Guide: The Prompt Framework Progression Matrix

## Recommended Presentation Order
When teaching beginners, you want to move from the most intuitive, human-centered framework to the most rigid, machine-centered architecture. 

1.  **CRAFT:** (The Visual Artist/Writer) — Focuses on presentation, style, and immediate layout.
2.  **PGTC:** (The Creative Director/Strategist) — Focuses on intent, hidden goals, and behavioral dynamics.
3.  **XML-Tagged:** (The Software Engineer) — Focuses on bulletproof structural consistency and programmatic parsing.

---

## The Rust Ringtone Blueprint: Framework Deep-Dive

Below is the comparative analysis of how each prompting framework handles the generation of your **Rust Audio Synthesizer Blueprint**.

### 🎨 1. The CRAFT Lens (Context, Role, Action, Format, Tone)
* **What it cares about:** The aesthetic surface layer and immediate presentation.
* **How it handles the Rust Blueprint:** 
    *   It effortlessly nails the polished, premium corporate workshop **Tone**.
    *   It ensures the output is cleanly laid out using standard Markdown headers (**Format**).
* **The Vulnerability:** It acts like a linear writer. While it makes the text look beautiful, it doesn’t natively force the AI to think about the *educational theory* underneath. It might easily forget to include the "AI-Assisted Prompt Matrix" because it's too focused on the outward-facing syllabus text.

### 🧠 2. The PGTC Lens (Persona, Goal, Task, Context)
* **What it cares about:** Hidden strategy, objective alignment, and motivation.
* **How it handles the Rust Blueprint:** 
    *   Because the **Persona** is an elite educator and the **Goal** is hidden AI literacy, PGTC guarantees that the **AI-Assisted Prompt Matrix** is explicitly generated.
    *   It ensures that the Rust project remains the *sandbox vehicle*, while the *true foreground skill* being taught is collaborating with an LLM.
* **The Vulnerability:** It focuses so heavily on the strategic intent that it might give the AI too much creative freedom with the visual formatting, resulting in inconsistent bullet points or paragraphs across different output runs.

### ⚙️ 3. The XML-Tagged Lens (Modular Code-Blocks)
* **What it cares about:** Machine readability, strict data constraints, and zero context drift.
* **How it handles the Rust Blueprint:** 
    *   By wrapping instructions in tags like `<hourly_schema>` and `<rust_mechanics>`, it programmatically forces the LLM to populate every single field.
    *   It ensures that Hour 4 and Hour 5 contain the exact same level of detail, sub-bullets, and rigorous technical breakdowns as Hour 1 without the model getting "lazy" or dropping constraints.
* **The Vulnerability:** It looks like computer code (`<system_role>...</system_role>`), making it visually intimidating and unintuitive for a room full of absolute beginners to write from scratch.
